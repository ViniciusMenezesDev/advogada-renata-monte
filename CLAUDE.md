# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

Static, single-page portfolio site for a lawyer (Renata Monte), published on GitHub Pages under the custom domain `renatamonte.com.br` (apex + `www`, both configured via DNS at Registro.br — see DNS records below). There is no build step, no package manager, and no framework: everything is plain HTML/CSS/JS plus CDN-hosted Bootstrap.

## Architecture

- `index.html` — the entire page. All sections (Navbar, Hero, Sobre, Áreas de Atuação, FAQ, Contato, Footer) live here as one file; there is no templating or componentization.
- `assets/css/style.css` — overrides Bootstrap via CSS custom properties defined in `:root` (`--color-primary`, `--color-secondary`, `--color-bg`, `--color-text`, `--font-heading`, `--font-body`). Change the palette/fonts here, not by editing individual rules.
- `assets/js/script.js` — minimal vanilla JS: sets the footer's current year and auto-collapses the mobile navbar after a nav link is clicked. Depends on the global `bootstrap` object from the Bootstrap bundle loaded in `index.html`.
- `assets/img/` — site images (profile photo, etc). Favicon-related files (`favicon.ico`, `favicon.svg`, `favicon-96x96.png`, `apple-touch-icon.png`, `site.webmanifest`, `web-app-manifest-*.png`) live at the **repo root**, generated by realfavicongenerator.net — don't move them into `assets/img`, the `<link>` tags in `index.html`'s `<head>` reference root-relative paths (`/favicon.svg`, etc).
- `CNAME` — GitHub Pages custom domain file, contains `renatamonte.com.br`. Required for the custom domain to keep working; don't delete it.
- `docker-compose.yml` — local preview only. Runs `nginx:alpine`, mounting the repo root read-only into `/usr/share/nginx/html`, exposed on `localhost:8080`.

External dependencies (all via CDN, no local copies): Bootstrap 5.3.3 (CSS + JS bundle), Bootstrap Icons 1.11.3, Google Fonts (Playfair Display, Montserrat).

## Commands

Local preview (no build needed — it's static HTML):

```bash
docker compose up -d      # serve the site at http://localhost:8080
docker compose down       # stop
```

There is no lint/test/build tooling in this repo.

## Content conventions

- All copy is in Portuguese (`lang="pt-BR"`).
- Practice areas are always mentioned in this order: **Direito Imobiliário** before **Registro de Marca** (applies to `<title>`, meta description/OG/Twitter tags, hero copy, "Sobre" copy, the Áreas de Atuação cards, FAQ order, and the footer line) — keep this order when adding or editing copy that lists both areas.
- Social sharing preview image: Open Graph / Twitter Card tags in `index.html`'s `<head>` point to an **absolute URL** (`https://renatamonte.com.br/assets/img/renata.jpg`); update `og:image`/`twitter:image` together if the photo changes, and keep `og:image:width`/`og:image:height` in sync with the actual file dimensions.
- Contact links are hardcoded in multiple places (navbar, hero, contact section, floating WhatsApp button, footer): WhatsApp `https://wa.me/5548984122537`, Instagram `https://www.instagram.com/renatamonte.adv/`, email `advogada@renatamonte.com.br`. Update all occurrences together if any of these change.

## Deployment

Push to `master` — GitHub Pages serves directly from the repo root on that branch (no Actions workflow). DNS: apex `renatamonte.com.br` uses GitHub Pages' 4 A records + 4 AAAA records; `www.renatamonte.com.br` is a CNAME to `viniciusmenezesdev.github.io`.
